# /review-article - 記事レビュー

`articles/` ディレクトリ内の記事を読み込み、構成・文章・訴求力・textlint の観点でレビューします。

コマンドに記事パスが含まれている場合は、そのファイルを優先して対象にします。

例:

```text
/review-article articles/2026-04-08-ai-exhibition-behind-the-scenes.md
```

## 実行手順

1. `.claude/skills/review-article/SKILL.md` を読み込む
2. コマンド引数に `articles/` 以下のファイルパスがあれば、それをレビュー対象にする
3. 引数がなければ SKILL.md に記載された手順で対象記事を特定する
4. `textlint` の自動修正以外は、必ずユーザー確認を取ってから適用する
5. SKILL.md に記載されたワークフロー（Phase 1〜4）を順番に実行する

## ワークフロー概要

```
Phase 1: 記事の読み込み
  → 引数のファイルパスがあればその記事を使う
  → なければ対象記事を質問して特定する
  → 必要に応じてレビューの重点を確認する

Phase 2: レビュー実行
  → review-checklist.md と article-template.md を参照する
  → 構成 / 文章ルール / 視点の一貫性 / 具体性 / 訴求力 / 行動経済学 / textlint を確認する
  → 総合評価 A / B / C を付けてレビューサマリーを作成する

Phase 3: フィードバック提示
  → 良い点と改善点をユーザーに提示する
  → すべて修正するか、選んで修正するか、修正しないかを確認する

Phase 4: 修正の適用
  → textlint の機械修正以外は、確認済みの改善点だけを記事へ反映する
  → textlint を再実行してエラー 0 を確認する
```

## 出力

- レビュー結果をテキストで提示する
- 修正を適用した場合は、更新後の記事を `articles/` 配下の元ファイルに保存する
- textlint チェック済みの状態で完了する
